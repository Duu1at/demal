# Тестирование с мок-данными

Этот документ описывает, как использовать мок-данные для тестирования функциональности верификации партнера.

## Структура моков

### ProfileRepositoryMockImpl
Мок-имплементация для `ProfileRepository`, которая позволяет тестировать:
- Получение статуса верификации партнера
- Обновление профиля партнера
- Обновление профиля пользователя

**Параметры:**
- `mockVerificationStatus` - Статус верификации для тестирования (PENDING, VERIFIED, REJECTED). Если `null` - симулирует отсутствие профиля (выбрасывает ошибку)
- `delay` - Задержка симуляции сетевого запроса (по умолчанию 500ms)

### UploadRepositoryMockImpl
Мок-имплементация для `UploadRepository`, которая симулирует загрузку файлов:
- Загрузка одного файла
- Загрузка множественных файлов
- Загрузка файлов с ключами

**Параметры:**
- `delay` - Задержка симуляции загрузки (по умолчанию 800ms)

## Использование

### Вариант 1: Использование main_mock.dart

1. Откройте `app/lib/main_mock.dart`
2. Запустите приложение через `main_mock.dart`
3. Для тестирования разных сценариев раскомментируйте нужный статус:

```dart
child: const AppMock(
  // Не указывать статус = профиль не существует (перенаправление на верификацию)
  // mockVerificationStatus: PartnerVerifyStatusEnum.pending,     // Заявка на проверке
  // mockVerificationStatus: PartnerVerifyStatusEnum.verified,    // Верифицирован
  // mockVerificationStatus: PartnerVerifyStatusEnum.rejected,    // Отклонен
),
```

### Вариант 2: Изменение импортов в main.dart

1. В `app/lib/main.dart` замените:
```dart
import 'package:app/app/view/app_view.dart';
```
на:
```dart
import 'package:app/app/view/app_view_mock.dart';
```

2. Замените `App()` на `AppMock()`:
```dart
child: const AppMock(
  mockVerificationStatus: PartnerVerifyStatusEnum.pending,
),
```

## Тестовые сценарии

### 1. Первый вход партнера (нет профиля)
```dart
child: const AppMock(
  // mockVerificationStatus не указан = null
),
```
- При попытке получить статус верификации выбрасывается ошибка
- Партнер автоматически перенаправляется на страницу верификации
- Форма будет пустой

### 2. Заявка на проверке (PENDING)
```dart
child: const AppMock(
  mockVerificationStatus: PartnerVerifyStatusEnum.pending,
),
```
- Будет показана форма верификации
- Будет отображен статус "Заявка на проверке"
- Можно обновить данные и отправить заявку повторно

### 3. Верифицированный партнер (VERIFIED)
```dart
child: const AppMock(
  mockVerificationStatus: PartnerVerifyStatusEnum.verified,
),
```
- Партнер будет автоматически перенаправлен на главную страницу
- Доступ ко всем функциям партнера (создание туров и т.д.)

### 4. Отклоненная заявка (REJECTED)
```dart
child: const AppMock(
  mockVerificationStatus: PartnerVerifyStatusEnum.rejected,
),
```
- Будет показана форма с комментариями администратора
- Статус "Заявка отклонена"
- Возможность повторной подачи заявки после исправления данных

## Логика перенаправления

Проверка статуса верификации происходит в `PartnerHomeView`:

1. При загрузке страницы проверяется статус верификации
2. Если статус `null` или не `VERIFIED` → перенаправление на `/partner/partner-verification`
3. Если статус `VERIFIED` → показывается главная страница партнера
4. Если произошла ошибка (профиль не существует) → перенаправление на страницу верификации

## Мок-данные

### Статус верификации
Мок возвращает следующие данные в зависимости от статуса:

- **null (не указан)**: Выбрасывает ошибку `Partner profile not found` → перенаправление на верификацию
- **PENDING**: "Заявка находится на проверке"
- **VERIFIED**: "Профиль успешно верифицирован"
- **REJECTED**: "Документы не соответствуют требованиям. Пожалуйста, загрузите более качественные фото."

### Загрузка файлов
Мок генерирует URL вида:
- Для изображений: `https://example.com/uploads/image/{timestamp}.{extension}`
- Для документов: `https://example.com/uploads/document/{timestamp}.{extension}`

## Примечания

- Все мок-запросы симулируют задержку сети (500-800ms)
- Моки не требуют реального API или интернет-соединения
- Для возврата к реальным данным просто используйте обычный `App()` вместо `AppMock()`
- **Важно**: При первом входе партнера (когда `mockVerificationStatus` не указан), мок выбрасывает ошибку, что симулирует отсутствие профиля и автоматически перенаправляет на страницу верификации
